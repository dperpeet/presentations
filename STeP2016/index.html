Bequemlichkeit

  • Cockpit: Wir probieren das Dinge nur einmal gemacht werden müssen
  • Ständig testen, Agile, jeder kann mitmachen
  • Review einfacher

Kathedrale vs. Basar

  • Kathedrale: Quellcode ist geschlossen oder nur mit jeder Version einsehbar
  • Entwicklung auf eine Gruppe beschränkt
  • Ein Chef/Architekt, ein Ziel: Dann fertig
  • Basar: Selbstorganisation, offen, organisch
  • Agile Prinzip
  • Viel mehr Overhead im Detail (s. Zellen mit DNS, Verschwendung?)
  • Code ist wegwerfbar, Evolution
  • Beitrag leisten: innerhalb von einer Woche wird es benutzt
  • Visuelle Darstellung
  • Sonst: Was ist mein Cluster?
  • Beispiel für Integration
  • Red Hat kombiniert die Technologien und macht sie zugänglich
  • Mit Tiefgang: nicht nur aufgesetzt, sondern auf allen Ebenen, um Benutzerinteraktion zu verbessern
